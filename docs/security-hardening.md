# Security Hardening

The stack is built defense-in-depth.

## Secrets

- All secrets via `.env` only (git-ignored; `generate-env.py` sets `chmod 600`
  on POSIX). `.env.example` ships invalid `CHANGE_ME_*` placeholders so a
  half-configured stack fails fast.
- **`ZITADEL_MASTERKEY`** encrypts secrets at rest — back it up separately from
  the DB; losing it makes encrypted data unrecoverable.
- The machine automation key lives only on the private `machinekey` volume.
- Terraform **state carries OIDC client secrets** → private backend, `*.tfstate`
  git-ignored.
- OIDC client secrets are generated by Zitadel and read via `tofu output` —
  never pre-shared or committed.

## Authentication

- **Federated users have no password** → IdP-only; no local-password attack
  surface for them.
- **`ForceMFALocalOnly`** enforces MFA for local accounts; Entra enforces MFA
  for federated logins.
- `IgnoreUnknownUsernames` avoids username enumeration; lockout policy limits
  brute force.

## Authorization

- App login is gated by **project authorization** — only users with a role grant
  may access an app.
- Synced grants are **namespaced** (`entra:`) and reconciled (add + remove) from
  Entra group membership.

## Network / transport

- TLS terminates at Traefik (`ExternalSecure=true`, port 443); internal traffic
  on the private `local` bridge. Zitadel h2c is only spoken inside the proxy.
- Graph app permissions are **read-only** and least-privilege (`User.Read.All`,
  `GroupMember.Read.All`, `ProfilePhoto.Read.All`).

## Supply chain / IaC

- Every image gets an **SBOM** in CI. Third-party GitHub Actions are tag-pinned;
  `bauer-group/*` reusable workflows use `@main`.
- The provision init container is **non-destructive**: it never runs an
  unattended destroy and never prunes UI-made resources.

## Logging

- No PII or secrets in logs (sync + backup log structured, redaction-safe
  messages).
